# LiberStudy UI 设计指南

> 版本: v0.1 | 日期: 2026-04-08 | 对应 PRD: v0.3
> 本文档为 UI 设计规范的完整参考，供 Figma 设计和前端开发使用。PRD 中只保留页面元素清单，详细规范见此文档。

---

## 1. 全局设计原则

**UI 设计工具**：Figma（配合 Figma MCP + Claude Code），设计稿作为前端开发直接依据。

**MVP UI 目标**：最简可验证全流程，不追求视觉精美，核心是功能闭环跑通。

**整体风格**：现代、极简、专业；"纸质"内容优先的无干扰感；扁平化图标，薄描边，克制的色彩使用。

**Figma 设计工作流**：
1. 先用 Figma 设计各页面线框图（Wireframe），在 Claude Code 中通过 Figma MCP 读取设计稿
2. 以线框图为基准生成前端代码
3. UI 评审后再出高保真视觉稿（V1.1 阶段），MVP 阶段以功能跑通为准

## 2. 页面布局规范

### 2.1 上课采集界面（场景②）

**布局**：三栏，与课后查看界面一致

| 栏位 | 宽度参考 | 内容 |
|------|---------|------|
| **左栏** | 可折叠，展开约 200px | 大纲导航：页码 + 页面标题列表；可折叠收起 |
| **中栏** | 弹性自适应 | PPT 画布；支持就地文本批注（见 §3）；顶部为 PDF 阅读器风格工具栏（见 §5） |
| **右栏** | 固定约 320px | 顶部：录音控制条（固定）；下方：当前页专属笔记区 |

**录音控制条**（右栏顶部固定）：开始 / 暂停继续 / 停止按钮 + 录音时长计时器

### 2.2 文件上传界面（场景①）

两个上传区域并排：PPT 文件区 / 音频文件区；支持拖拽和点击选择；两个文件均上传成功后"开始处理"按钮激活。

### 2.3 处理进度界面

线性进度条 + 当前阶段文字说明（语音转录中 / PPT 内容解析中 / 翻页时间轴构建中 / 笔记生成中）+ 预估剩余时间。

**笔记生成阶段细化**：进入「笔记生成中」阶段后，进度文字切换为逐页计数格式：`第 3/18 页笔记生成中...`；每页生成完成后立即在笔记查看界面渲染该页笔记（流式逐页展示，无需等待全部完成）。用户可在笔记仍在生成时提前查看已完成的页面。

### 2.4 课后笔记查看界面

**布局**：三栏

| 栏位 | 宽度参考 | 内容 |
|------|---------|------|
| **左栏** | 可折叠，展开约 200px | 大纲导航：Hover 浮出缩略图，点击跳转页面 |
| **中栏** | 弹性自适应 | PPT 主画布；展示就地文本批注标记；顶部为 PDF 阅读器风格工具栏（见 §5） |
| **右栏** | 固定约 320px | 顶部：药丸型笔记切换控件（见 §4）；下方：当前页专属笔记内容 |

## 3. 就地文本批注（PPT 画布交互）

参考 Edge 浏览器 PDF 文件中"添加文本"功能：

| 交互步骤 | 行为 |
|---------|------|
| 用户点击中栏 PPT 画面任意位置 | 就地弹出可输入文字框，位置与点击坐标对齐 |
| 用户键入文字并确认（Enter / 点击外部） | 文字框收起，以轻量标注样式留在 PPT 画面对应位置 |
| 系统自动记录 | 页码 + 当前录音相对时间戳 + Y 轴位置 |
| 右栏同步 | 该批注即时出现在右栏当前页笔记区 |
| 右栏直接输入 | 用户也可在右栏笔记区点击输入，与中栏就地批注双向实时同步 |
| 翻页 | 中栏画面只显示当前页批注；右栏自动切换为新页专属笔记内容 |

**降级策略**（开发周期紧张时）：优先实现基础版（点击弹出固定文字框，Y 轴位置记录）；缩放同步和流畅动画 V1.1 补全。

## 4. 药丸型笔记切换控件（课后查看界面右栏顶部）

参考 Granola 风格的 Pill-shaped Segmented Control：

### 4.1 容器

| 属性 | 规格 |
|------|------|
| 形状 | 药丸形，圆角柔和 |
| 背景色 | 暖灰色 / 米色（Off-white / Cream） |
| 边框 | 无边框，营造"纸质"无干扰感 |
| 尺寸 | 紧凑，适合放置于右栏顶部工具区 |

### 4.2 左分段：我的笔记

| 属性 | 规格 |
|------|------|
| 图标 | 深灰色列表图标（三条水平线，lucide-react `List`）|
| 文字标签 | "我的笔记" |
| 激活态背景 | 与容器相同（暖灰色，表示非活动默认态）|
| Hover 提示 | 鼠标悬浮时，控件底部浮出小提示文字 **"My note"** |

### 4.3 右分段：AI 笔记

| 属性 | 规格 |
|------|------|
| 激活态背景 | 纯白色，带微妙阴影 / 轻微凸起效果 |
| 图标组合 | ✨ 星星簇图标（一大两小四角星，lucide-react `Sparkles`）+ "AI 笔记" 文字 + 向下雪佛龙（`ChevronDown`） |
| 点击雪佛龙 | 展开 AI 笔记模板下拉菜单（不切换分段） |

### 4.4 下拉菜单（AI 笔记模板）

| 样式属性 | 规格 |
|---------|------|
| 背景 | 半透明干净卡片，subtle shadow |
| 触发方式 | 点击右分段的雪佛龙图标 |
| 当前选中 | 模板名称前显示勾选状态 |

**4 种模板**：

| 模板名称 | 适用场景 | 生成逻辑 |
|---------|---------|---------|
| **基于我的笔记扩写** | 有用户文本批注的页面 | 用户批注关键词/疑问 + 对应时段精校转录稿 → LLM 扩写为完整结构化段落；无批注的页面此项灰显 |
| **全 PPT 讲解笔记** | 所有页面 | PPT 每页 bullet + 对应精校转录稿 → 行级 LLM 对齐注释 + page_supplement 补充说明（被动学习基础产物）|
| **完整综合笔记** | 所有页面 | PPT 内容 + 老师讲解 + 用户批注三合一，按页生成最完整版本；有批注的页面叠加主动学习扩写 |
| **大纲摘要** | 全课回顾 | 跨所有 PPT 页面，提取核心概念、关键论点和重要例证，生成一页式全课概览；适合快速复习 |

### 4.5 简单 / 详细粒度切换

每种模板均提供顶部粒度切换开关：

| 模式 | 行为 |
|------|------|
| **简单** | 精炼概括，每个知识点 1-2 句，控制笔记长度 |
| **详细** | 尽量保留老师讲解细节和例证，允许更长段落 |

两种模式通过 Prompt 工程控制 LLM 输出密度，共享同一份精校转录稿，不影响 ASR 处理成本。

### 4.6 交互逻辑

- 左右分段互斥切换，切换动画平滑过渡
- "AI 笔记"侧雪佛龙点击触发模板下拉，不切换分段
- 模板切换 / 粒度切换触发当前页笔记重新渲染（如已缓存则不重新调用 LLM）
- 字体：标签/标题使用清晰无衬线体

## 5. PDF 阅读器风格顶部工具栏（中栏画布顶部）

参考 Edge 浏览器 PDF 阅读器的工具栏风格。图标库统一使用 `lucide-react`，描边粗细 `strokeWidth={1.5}`，图标尺寸 `w-5 h-5`。

### 5.1 全局样式

| 属性 | 规格 |
|------|------|
| 容器 | `w-full flex items-center px-3 h-12` |
| 背景色 | 极浅灰（`#F9FAFB` / Tailwind `bg-gray-50`）|
| 下边框 | `border-b border-gray-200` |
| 字体 | `text-sm`，清晰无衬线体 |
| 图标颜色 | 深灰 `text-gray-700` |
| Hover 效果 | `hover:bg-gray-200 rounded p-1.5 transition-colors cursor-pointer` |
| 分隔符 | `<div className="w-px h-5 bg-gray-300 mx-2" />` |

- 右对齐分组用 `ml-auto` 推至最右侧

## 7. 颜色规范（MVP 阶段）

| 用途 | 颜色 |
|------|------|
| 页面背景 | `bg-white` / `bg-gray-50` |
| 工具栏背景 | `bg-gray-50` (`#F9FAFB`) |
| 药丸控件容器 | Off-white / Cream（近似 `#FAF9F7`）|
| AI 笔记激活态 | `bg-white` + subtle shadow |
| 边框 / 分隔线 | `border-gray-200` / `bg-gray-300` |
| 主文字 | `text-gray-900` |
| 次要文字 / 提示 | `text-gray-500` |
| 图标 | `text-gray-700` |
| Hover 背景 | `hover:bg-gray-200` |
| 警告标识 ⚠️ | 系统警告黄，用于对齐置信度低的页面 |

### 7.1 笔记内容字色区分规范

笔记查看界面右栏中，用户笔记与 AI 生成内容通过字色区分，不使用背景色块。

| 内容类型 | 颜色规格 | Tailwind 类名 | 说明 |
|---------|---------|--------------|------|
| **用户原始笔记** | `#000000` | `text-gray-950` / `text-black` | 纯黑，代表「我写的」，视觉权重最高 |
| **AI 生成内容** | `#374151` | `text-gray-700` | 深灰，有明显区分但不影响可读性；**禁止使用 `text-gray-400` 或更浅的颜色** |

> **设计意图**：颜色差异传达「来源区分」而非「重要性差异」，AI 内容不应显得"次要"，只需与用户自己写的笔记有视觉可辨别的差异。

## 8. 导出规范

### 8.1 导出内容原则

导出内容为**纯笔记文字**，不嵌入 PPT 截图。以 PPT 页码和页面标题作为章节锚点，方便在 Notion / Obsidian 中直接导航。

### 8.2 Markdown 导出结构

```markdown
# {PPT 文件名} 课堂笔记
> 生成时间：{日期}

## 第 1 页：{页面标题}

- {bullet 原文}
  - {AI 行级注释} [00:03:21]
- {bullet 原文}
  - {AI 行级注释} [00:05:47]

**补充说明**
{page_supplement 内容}

> 💡 **[我的笔记]**
> {用户原始批注文字}

---

## 第 2 页：{页面标题}
...
```

**格式规则**：
- 每页以 `## 第 N 页：{标题}` 作为 H2，页间用 `---` 分隔
- AI 生成内容（bullet 注释、补充说明）正常输出，代码注释标记 `<!-- AI -->` 供工具识别
- 用户笔记统一用 `> 💡 **[我的笔记]**` 块引用格式，视觉上与 AI 内容区分
- 时间戳格式：`[HH:MM:SS]`，紧跟注释文字后

### 8.3 PDF 导出

与 Markdown 结构一致，不含 PPT 截图，前端使用 jsPDF + html2canvas 实现。

### 8.4 一键复制单页笔记

- 位置：右栏笔记区顶部工具行，放置复制图标按钮（`lucide-react Copy`，`w-4 h-4`）
- 复制内容：当前页的完整笔记文字（Markdown 格式），包含 AI 注释 + 用户笔记
- 点击后显示短暂的「已复制」Toast 提示（1.5 秒自动消失）
