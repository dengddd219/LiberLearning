# Expert2_Kano 分析报告：LiberStudy 需求挖掘与优先级排序

## 数据概览
- **数据来源**：12个关键词，237条帖子，1501条评论
- **覆盖维度**：恐惧与痛点（Fears & Pain Points）、核心需求与期望（Needs & Expectations）、行为习惯与认知（Behaviors & Cognition）
- **核心发现**：用户最核心的痛苦是"听"与"记"不同步导致认知过载，以及现有AI工具无法将**PPT页面**与**讲解内容**自动对齐，导致课后整理笔记时需要大量手工操作。

---

## 1. 核心发现（5条）

### 发现一：转录工具是标配，但"仅转录"价值有限
用户已普遍使用飞书妙记、通义听悟、夸克AI听记等工具完成音视频转文字，但大量评论反馈仅转文字后内容仍需大量手工整理（"导出TXT后还要自己重新修改错别字、整理段落"、"整理完还要熬夜做PPT课件"）。**转录是基础设施，对齐与结构化才是差异化价值**。

### 发现二：PPT场景存在独特痛点——"对着PPT补笔记崩溃"
这是数据中极为突出的原生场景关键词。用户反馈集中在：
- 课堂/会议PPT翻页快，边听边拍照/截图后，课后无法将笔记与具体PPT页对应
- 一个多小时课程，整理笔记要再花同等时间
- 现有工具（飞书妙记、通义等）只能转文字，无法解决"哪段话对应哪一页PPT"的问题

这是现有工具**完全没有覆盖**的核心场景，也是 LiberStudy 最强的切入点。

### 发现三：自动生成笔记质量不足，用户仍需二次加工
AI生成的结构化笔记被频繁吐槽："AI味一眼就能看出来"、"关键信息丢失率很高"、"要么狗屁不通，要么很平很干"。但用户愿意接受"AI初稿+人工快速校对"的工作流，这说明笔记生成的**易编辑性**比一次性完美更重要。

### 发现四：用户已自发形成"工具组合工作流"
大量评论显示用户将多个工具组合使用（飞书妙记+Kimi、飞书妙记+DeeSeek、通义+ChatGPT等），说明单工具无法满足需求，用户在主动寻找"转录+整理+结构化"的一站式方案。LiberStudy 有机会成为这个一站式方案。

### 发现五：移动端与访问限制是痛点，而非功能
飞书妙记的免费时长限制（每月300分钟）、部分工具无法在手机端修改文字、无法导出等限制被大量抱怨。用户在意的不是功能多寡，而是"随手能用、随时可改、导出无障碍"的轻便体验。

## 2. 需求清单与 Kano 分类

| ID | 需求描述 | Kano分类 | 典型证据 |
|----|---------|---------|---------|
| R01 | **PPT页面与讲解内容自动对齐**（双栏联动） | 基本需求 | "对着PPT补笔记崩溃"下大量帖子；"能不能出一款能自动对齐PPT页面的笔记工具"类评论 |
| R02 | **音视频转文字（ASR）** 准确率高 | 基本需求 | 通义/飞书妙记/夸克的使用分享；"转录准确率"是评论区最高频提问 |
| R03 | **自动结构化笔记生成**（要点/章节/关键词提取） | 期望需求 | "导出的文字稿还要再花时间整理"；Kimi+飞书妙记组合工作流的广泛使用 |
| R04 | **快速定位与浏览**（时间戳跳转、思维导图概览） | 期望需求 | "点击某个模块可以直接跳转到该部分内容"；"生成思维导图帮助厘清思路" |
| R05 | **多格式导出**（TXT/MD/Word/PPT） | 期望需求 | "导出txt方便后续操作"是各工具介绍帖标配；"超过10分钟的语音转的文字不能编辑排版"被吐槽 |
| R06 | **发言人识别与标注** | 期望需求 | "能不能标注发言人"类评论；"不同人说话能自动区分"是会议场景刚需 |
| R07 | **多语言/方言支持**（至少中文） | 基本需求 | 非英语/非中文（如韩语）的支持准确率极低；Granola中文支持差被广泛吐槽 |
| R08 | **边听边做标记/批注**的实时辅助 | 兴奋需求 | 直播/会议中"一边同步转文字一边补充自己想法"的工作流 |
| R09 | **课后AI问答**（基于课程内容追问） | 兴奋需求 | "基于讲稿上下文提供解释，确保知识连贯性"；"随时追问难点" |
| R10 | **跨设备同步与长期存储**（知识库积累） | 兴奋需求 | "笔记越来越多，找不到以前的资料"；"夸克的多端同步"被反复强调 |
| R11 | **实时转录**（低延迟） | 基本需求 | "能不能实时转文字"、"延迟非常鸡肋"被频繁提及 |
| R12 | **多人协作/共享**（session URL分享） | 兴奋需求 | "协作与讨论"；PRD中已规划无需账号体系，session URL替代 |
| R13 | **移动端体验流畅**（手机/iPad可用） | 期望需求 | "手机端不能修改文字"被大量吐槽；"电脑端数据不能同步到手机端" |
| R14 | **隐私保护与数据安全**（文件24h后删除） | 基本需求 | PRD已规划；数据丢失/隐私担忧是笔记工具的隐性基本需求 |
| R15 | **免费额度充足**（或合理定价） | 基本需求 | "免费时长只有300分钟根本不够"；"收费太贵" |

## 3. 优先级矩阵结论

### 矩阵框架（频次 × 情感强度）

| | **高频** | **中频** | **低频** |
|---|---|---|---|
| **高情感** | R01(PPT对齐)、R02(ASR准确率) | R03(结构化笔记)、R04(快速定位) | R08(边听边标记) |
| **中情感** | R07(多语言支持) | R05(多格式导出)、R06(发言人识别) | R09(AI问答) |
| **低情感** | R13(移动端流畅) | R14(隐私安全)、R15(合理定价) | R10(知识库积累)、R12(多人协作) |

### 优先级结论

**最高优先级（必做，否则用户直接放弃）：**
- **R01：PPT页面与讲解内容自动对齐** — 频次最高、情感最强，是 LiberStudy 区别于所有竞品（飞书妙记/通义/Granola/NotebookLM）的核心差异点
- **R02：ASR转录准确率** — 基本功，但必须及格（至少92%+）；低于此阈值用户直接弃用

**高优先级（显著提升留存）：**
- **R03：自动结构化笔记生成** — 用户最愿意为"省时间"付钱的功能
- **R04：时间戳快速定位与思维导图概览** — 复习场景刚需，显著提升产品使用频率

**中优先级（差异化加分）：**
- R05/R06（导出与发言人）：是可用性保障，但非核心差异化
- R13（移动端）：MVP后需尽快补齐，学生场景移动端使用率极高

**低优先级（MVP可后做）：**
- R08/R09（边听边标记/AI问答）：属于兴奋需求，MVP后期迭代
- R10/R12（知识库/协作）：非学生首周的刚性需求

## 4. PRD 覆盖率分析

### PRD 六大核心功能对照

| PRD功能 | 覆盖评级 | 分析 |
|---------|---------|------|
| **P0-1: PPT/PPTX + MP4/WebM/MP3/WAV 上传** | ✅ 完全覆盖 | 各关键词下大量帖子涉及视频/音频上传场景，需求真实且高频 |
| **P0-2: 幻灯片检测（帧分析）** | ✅ 完全覆盖 | "对着PPT补笔记崩溃"原生场景；用户无法手工对齐PPT页面是核心痛点 |
| **P0-3: ASR转写（带时间戳）** | ✅ 完全覆盖 | 飞书妙记/通义/夸克的使用分享证明需求成熟；但现有工具仅转文字，缺乏与PPT对齐能力 |
| **P0-4: Multimodal对齐（Transcript↔PPT）** | ⚠️ **PRD有，但竞品空白，是最大差异化** | **这是PRD六大功能中唯一没有竞品完整实现的功能**，也是用户最期待的核心价值 |
| **P0-5: 笔记生成（LLM结构化）** | ⚠️ 覆盖但深度不足 | 现有工具（Kimi+飞书/通义）已做初版；但"AI味重"、"关键信息丢失"说明生成质量需重点提升 |
| **P0-6: 双栏查看（Slide Nav + Notes）** | ✅ 完全覆盖 | 搜索关键词"音视频 双栏 笔记 软件"有162条评论，为数据集中最高；用户已有明确的产品形态认知 |

### PRD 遗漏的关键需求（需补充到PRD）

1. **R06 发言人识别**：PRD完全未提及，但会议/课堂场景必需
2. **R07 多语言支持边界**：PRD说"MVP单语言"，但数据表明即使是中文，不同方言（四川话等）也会影响准确率，需明确技术边界
3. **R05 多格式导出**：PRD完全未提及，但用户明确需要TXT/MD/Word/PPT导出；且"导出后仍可编辑"比"一次性生成PPT"更重要
4. **R08 实时边听边标记**：Granola的精髓是"会议中AI转录、用户同步补充想法"，PRD的"双栏查看"是否支持实时标注需明确

## 5. MVP 风险项（P0/P1/P2）

### P0 风险项（不做用户直接放弃）

| 风险项 | 描述 | 应对策略 |
|--------|------|---------|
| **ASR准确率不达标** | 若转录准确率低于90%，用户直接弃用转向飞书妙记/通义 | MVP必须接入 Whisper（中文优化）或自研ASR，优先保证普通话准确率 |
| **PPT对齐失败率过高** | 若PPT与Transcript对齐准确率低于80%，双栏功能形同虚设 | 技术方案：OpenCV帧差法+CLIP fallback（PRD已规划），需实测达标后再上线 |
| **文件上传限制过严** | PRD限制120分钟/视频，若用户课程经常超过2小时，体验断裂 | 考虑分段落处理或明确告知用户分段方式 |
| **导出后无法编辑** | 用户反馈最强的需求是"可编辑"，而非"一次性完美生成" | MVP笔记输出优先保证可复制/可编辑，AI生成仅作为初稿 |

### P1 风险项（不做用户留存率显著下降）

| 风险项 | 描述 | 应对策略 |
|--------|------|---------|
| **移动端体验缺失** | 学生场景中手机/iPad是主要学习设备，PC端不够 | MVP至少保证移动端可查看（编辑可PC端），避免"只能在电脑用"的竞品式抱怨 |
| **中文夹杂英文场景支持差** | 课程中出现专业术语/英文PPT时，ASR和PPT OCR双重困难 | MVP明确支持中英混合场景 |
| **生成笔记AI味太重** | "要么狗屁不通，要么很平很干"是现有AI笔记工具的统一槽点 | 提示词工程重点优化；参考Granola的"用户主导+AI辅助"思路 |

### P2 风险项（不做不影响初期留存，但影响口碑传播）

| 风险项 | 描述 | 应对策略 |
|--------|------|---------|
| **多人协作/分享能力弱** | PRD以session URL替代账号，但分享后的协作体验未规划 | v1.1规划：查看/批注/评论功能 |
| **知识库积累缺失** | 用户习惯将课程笔记长期保存、多节课关联查看 | v1.2规划：课程文件夹/知识库体系 |
| **定价超过用户预期** | 参考Granola $18/月被质疑"赔钱做"，用户对AI工具付费意愿存在但敏感 | MVP免费或极低定价（参考通义免费策略），后期按调用量计费 |

## 附录：PRD 补充建议

基于 Kano 分析，建议在 PRD 中增加或明确以下内容：

1. **发言人识别**纳入P0功能，或明确第一版不支持（避免用户预期错位）
2. **导出格式**明确为TXT/MD优先，PPT导出为次优先级
3. **移动端战略**：明确MVP是"查看优先"还是"全功能"，避免"PC only"陷阱
4. **技术指标承诺**：ASR准确率（普通话≥92%）、对齐准确率（≥85%）、响应时间等应在PRD中量化
5. **Granola式"实时标注"功能**：考虑在双栏界面中支持用户在会议/课程进行中同步打字补充想法（Granola周留存70%的核心原因）
