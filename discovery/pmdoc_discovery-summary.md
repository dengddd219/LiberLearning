## Phase 1: 问题构建

### 问题假设

我们相信**大学生**在课后回看PPT课程视频时效率极低（2-3倍原始时长），因为现有工具只能处理单一信息流（纯转录或纯PPT），无法将讲者口语讲解与PPT页面内容做结构化对齐，导致用户不得不反复拖动进度条，整理出的笔记仍然碎片化、不可检索。

### 信心水平

| 假设 | 信心 | 来源 |
|------|------|------|
| 课后回看效率低是真实痛点 | **高** | 个人经验 + 社区观察（知乎/小红书/B站） |
| PPT锚点式笔记比全文AI总结更好 | **低** | 产品直觉，未验证 |
| 用户愿意上传PPT+视频两个文件 | **低** | 未验证，可能操作太重 |
| 用户需要"结构化笔记"而非"转录+总结" | **中偏低** | 竞品分析推断 |
| 985/211理工科学生是核心用户 | **中** | 个人经验 |

### 待验证的 3 个核心研究问题

**RQ1: PPT锚点 vs 全文总结**
- 用户复习时的心智模型是什么？是按"这一页讲了什么"来回忆，还是按"这节课讲了哪些主题"？
- 如果给用户两种笔记格式，他们更倾向哪种？

**RQ2: 上传行为意愿**
- 用户课后手里通常有哪些素材？（PPT？录屏？录音？拍照？）
- 同时上传PPT+视频是否过于繁琐？什么程度的操作成本是可接受的？

**RQ3: 结构化深度需求**
- 用户对AI生成笔记的期望是什么？只要关键要点提炼，还是要完整的知识结构？
- "飞书妙记/Otter式纯转录+总结"在哪些场景不够用？具体不够用在哪里？

### 成功/失败标准

- **假设成立信号**: 5个访谈对象中≥3人表达"我复习时确实按PPT页面来回忆/查找"
- **假设失败信号**: 多数用户表示"有个全文总结就够了，不需要按页面拆分"
- **需要pivot的信号**: 用户的核心痛点不在"回看"，而在别的环节（比如"根本没时间回看"）

## Phase 2: 桌面研究计划

### 研究目标

在用户访谈前，通过竞品深度分析和社区数据收集，缩小假设范围，识别最关键的1-2个假设进入轻量访谈验证。

### 桌面研究任务

**任务1: 竞品体验对比**（预计2-3小时）
- 实际体验飞书妙记、通义听悟、Otter.ai、NotebookLM的笔记生成效果
- 重点观察：它们的输出格式、结构化程度、是否做PPT对齐
- 记录：用同一段课程视频测试，对比各家输出

**任务2: 社区痛点挖掘**（预计2-3小时）
- 知乎搜索：课后回看、课堂笔记、AI笔记 等关键词
- 小红书搜索：课堂录屏、AI学习工具、考试复习方法
- B站弹幕/评论：网课视频的用户吐槽
- 重点提取：用户原话中的痛点表述、现有workaround、对AI工具的评价

**任务3: 用户行为数据**（预计1小时）
- 课程录屏/录音APP的应用商店评论（录音宝、讯飞听见等）
- 笔记类APP评论中关于"课堂"场景的反馈
- B站网课视频完播率数据（如有公开数据）

### 预期产出

- 竞品能力矩阵（输出格式/结构化程度/PPT对齐能力/用户评价）
- 社区痛点频次统计（哪些痛点被提及最多）
- 用户原话语料库（用于后续访谈设计和产品文案）

## Phase 2: 桌面研究结果

### 竞品能力矩阵

| 能力维度 | 飞书妙记 | 通义听悟 | Otter.ai | NotebookLM | 讯飞听见 | **LiberStudy** |
|---------|---------|---------|----------|------------|---------|---------------|
| 语音转写 | Yes | Yes | Yes (仅英文) | No | Yes | Yes |
| AI 摘要/总结 | Yes | Yes (强) | Yes | Yes (强) | 基础 | Yes |
| 章节自动分段 | Yes | Yes | Yes | Yes | No | Yes (PPT页粒度) |
| **PPT页面级时间轴对齐** | 部分 (仅飞书会议) | 部分 (按时间点切分) | 部分 (仅实时会议截屏) | No | 部分 (企业方案) | **核心能力** |
| PPT视觉内容解析(图/表/公式) | No | No | No | 有限 | No | **核心能力** |
| 口语降噪→书面化重写 | No | No | No | N/A | No | **核心能力** |
| 逐页结构化知识笔记 | No | No | No | No | No | **核心能力** |
| 支持离线上传视频+PPT | No | Yes | No | N/A | Yes | **Yes** |
| 中文支持 | Yes | Yes | No | Yes | Yes | Yes |

### 竞品关键发现

1. **PPT对齐能力普遍"浅"**：飞书妙记、通义听悟、Otter.ai都有某种程度的幻灯片-时间轴对齐，但均停留在"截屏/时间点标记"层面，不解析PPT页面内的视觉内容（图表、公式、关键术语）
2. **"口语转书面"无人做**：所有竞品的转录输出都保留大量口语噪音，没有产品做"口语讲解与PPT内容融合后输出可读性高的书面笔记"
3. **"逐页结构化笔记"是空白地带**：现有竞品输出要么是全文逐字稿，要么是全文AI摘要，没有产品以PPT单页作为信息组织的基本单元
4. **离线场景被忽视**：Otter.ai和飞书妙记仅支持实时会议，不支持"事后上传视频+PPT"，而这是大学生课后复习的主流场景
5. **通义听悟是最接近的竞品**：支持上传、有PPT时间点切分、免费额度最多，但输出仍不够结构化

### 社区痛点频次统计

| 排名 | 痛点 | 频率 | 说明 |
|------|------|------|------|
| 1 | 录屏/录音回看时间成本极高 | ~35% | "90分钟课需2-4小时回看整理，2倍速都看不完" |
| 2 | 听课与记笔记无法兼顾 | ~25% | "来不及记"、"顾此失彼"、"记了前面忘后面" |
| 3 | 录音/转写与PPT无法对齐 | ~18% | 不知道录音某段话对应哪页PPT，手动对齐极其耗时 |
| 4 | AI转写工具专业术语识别差 | ~13% | 理工科/医学/法学专业词汇错误率高 |
| 5 | "录了不看"——录屏吃灰 | ~9% | 因回看成本太高导致行为断裂 |

### 用户原话语料（10条关键表述）

1. "一节90分钟的课，我课后光听录音整理笔记就要花3个多小时，比重新上一遍课还累。"
2. "边听课边记笔记根本顾不上，记了前面的，后面老师说什么完全没听进去。"
3. "录了一学期的课堂录音，到期末发现一条都没听过，全在吃灰。收藏即学会，录音即复习。"
4. **"最痛苦的是回听录音的时候找不到老师当时讲到哪页PPT了，只能一边拖进度条一边翻课件，来回切换窗口快疯了。"**
5. "用了讯飞听见转写课堂录音，专业术语错得离谱，'高斯分布'给我识别成'搞死分布'。"
6. "通义听悟的总结太笼统了，没有重点，一节课给我总结成三段话，完全不知道哪里是考点。"
7. **"期末复习就是对着PPT一页一页翻，翻到某页想不起来老师讲了什么，又得去翻录音，一个晚上只能搞完一门课。"**
8. "飞书妙记对学生不太友好，本来是给企业会议用的，功能入口不直观，免费额度也不够一学期用。"
9. **"我现在的流程是：上课拍PPT→录音→课后打开PPT和录音一起听→手动在每页PPT旁边写注释。一门课整理下来要整整一个下午。"**
10. "数学公式、化学方程式这些AI根本转写不出来，只能靠拍PPT截图，但截图又和文字笔记分开放，复习的时候到处找。"

### 行为模式关键发现

**用户的复习心智模型是"按PPT页面"，而非"按主题/全文"。**

典型复习流程：下载PPT → 逐页翻PPT → 翻到不懂的回去找录音 → 手动在PPT旁边写注释。"翻PPT式复习"在小红书上是高频内容类型。

| 现有Workaround | 采用率 | 核心缺陷 |
|---------------|--------|----------|
| PPT+录音手动对照 | 最高 | 极其耗时，切换窗口/拖进度条 |
| iPad上PPT标注(GoodNotes/Notability) | 中高 | 仍需手动整理，且录音不同步 |
| AI转写工具(讯飞/通义) | 中 | 只有文字流，与PPT脱节；专业词不准 |
| 上课拍PPT照片 | 高 | 碎片化严重，照片和笔记分散各处 |

### 桌面研究结论：假设信心更新

| 假设 | 研究前信心 | 研究后信心 | 变化原因 |
|------|-----------|-----------|---------|
| 课后回看效率低是真实痛点 | 高 | **很高** | 社区数据强力验证，Top 1痛点 |
| PPT锚点式笔记比全文AI总结更好 | 低 | **中偏高** | 行为模式显示用户确实按PPT页面复习；竞品全文总结被吐槽"太笼统" |
| 用户愿意上传PPT+视频两个文件 | 低 | **中** | 用户现有流程已经在同时使用PPT+录音，上传两个文件符合习惯；但仍需访谈验证 |
| 用户需要"结构化笔记"而非"转录+总结" | 中偏低 | **中偏高** | 用户明确吐槽通义听悟"总结太笼统，不知道哪里是考点"；需要逐页粒度的深度内容 |
| 985/211理工科学生是核心用户 | 中 | **中** | 痛点表述中理工科确实突出（公式/图表），但文商科也有强需求；需访谈确认 |

**最关键待验证假设（进入访谈阶段）：**
1. ~~PPT锚点 vs 全文总结~~ → 桌面研究已初步验证（用户按PPT页面复习），但仍需访谈确认
2. **用户上传意愿和操作成本容忍度** → 桌面研究无法回答，必须通过访谈验证
3. **结构化深度需求** → 需要了解用户对"每页笔记应该包含什么"的具体期望
