# LiberStudy MVP-0 Implementation Plan (Superpowers Breakdown)

> 日期: 2026-04-09 | 状态: Draft | 对应 Spec: mvp-architecture-design.md | 对应 PRD: v0.4

---

## 总览

基于 PRD v0.4 和已确认的技术架构 Spec，将 MVP-0 开发拆解为 **6 大阶段（Phase）**，每个阶段下拆为 **具体任务（Task）**，每个任务再往下拆一层为 **子任务（Sub-task）**。

两阶段开发策略：**阶段 A（Mock 数据跑通界面）→ 阶段 B（接入真实 API）**。

验证路径：`/session` → `/processing` → `/notes/:session_id`

---

```
        ↓

  ├─ Task 2.1 Mock 数据  ━━ 先做
  ├─ Task 2.2 采集界面   ━━ 核心（最复杂）
  ├─ Task 2.4 笔记查看   ━━ 核心
  ├─ Task 2.3 处理进度   ━━ 较简单
  ├─ Task 2.5 文件上传   ━━ P1
  └─ Task 2.6 中断恢复   ━━ 可后做
        ↓
Phase 3（阶段 A 联调）  ━━ 联调 + 导出
        ↓
Phase 4（阶段 B，真实 API）  ━━ 重点阶段
  ├─ Task 4.1 PPT 解析   ━━ 先做（无外部 API 依赖）
  ├─ Task 4.2 音频处理   ━━ 先做（无外部 API 依赖）
  ├─ Task 4.3 ASR 转录   ━━ 需 API Key
  ├─ Task 4.4 语义对齐   ━━ 需 API Key
  ├─ Task 4.5 LLM 生成   ━━ 需 API Key
  └─ Task 4.6 串联测试   ━━ 最后做
        ↓
Phase 5（埋点）  ━━ 可与 Phase 4 并行
        ↓
Phase 6（部署上线）
```

## 关键决策点（需要在开发中确认）

1. **Tailwind CSS 版本**：v3 还是 v4？（v4 配置方式有变化）
2. **IndexedDB 库选择**：推荐 `idb`（轻量 Promise 封装）
3. **音频播放器**：自建还是用 `react-h5-audio-player`？（推荐自建，控制力更强）
4. **Railway Dockerfile**：需要预装 LibreOffice + FFmpeg，建议基于 `python:3.11-slim` + apt-get 安装
5. **ASR API Key 申请**：阿里云 ASR 和 OpenAI API Key 需提前准备
