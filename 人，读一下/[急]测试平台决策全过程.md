# LiberStudy 策略决策记录

> **用途**：记录你在测试平台上验证过的每一个策略决策。  
> 测试平台（Streamlit）是"试验田"，这个文件是"决策档案"，后端生产代码是"最终落地"。  
> 每条记录回答：**我在哪个环节用了什么策略、为什么、对应代码在哪里**。

---

## 如何使用这个文件

1. **在测试平台上验证完一个策略** → 在对应章节补一条记录
2. **记录三件事**：是什么 / 为什么选它 / 对应哪个文件+参数
3. **状态标记**：`[测试中]` `[已定稿]` `[已废弃]`
4. **最终要做的事**：所有条目都变成 `[已定稿]` 后，你就有了完整的生产策略方案

## 模块 4：语义对齐（Alignment）

**对应文件**：`backend/services/alignment.py`

### 4.1 核心算法

三路信号融合：
1. **ASR × PPT 语义相似度**（主信号）—— OpenAI text-embedding-3-small cosine similarity
2. **用户标注锚点**（强覆盖）—— 用户手动标记"这段话对应第 N 页"
3. **时间轴顺序**（软约束）—— 老师通常顺序讲，但允许回翻

### 4.2 关键参数（所有参数在文件顶部常量区）

| 参数名 | 当前值 | 含义 | 状态 |
|--------|--------|------|------|
| `OFF_SLIDE_THRESHOLD` | 0.30 | 低于此相似度 → 判定为"离页内容"，进入 page_supplement | [测试中] |
| `LOW_CONFIDENCE_THRESHOLD` | 0.60 | 低于此值 → UI 显示 ⚠️ 低置信警告 | [测试中] |
| `PAGE_SWITCH_K` | 3 | 连续 K 个片段指向新页才触发翻页（防抖） | [测试中] |
| `OFF_SLIDE_UPGRADE_THRESHOLD` | 0.60 | 离页片段与上一对齐页相似度超此值 → 归入该页 | [测试中] |

**如何调整**：直接修改 `alignment.py` 顶部的常量值。测试后在此记录新值和原因。

**待决策**：
- [ ] OFF_SLIDE_THRESHOLD 最优值（当前 0.30 是否太宽/太严？）
- [ ] PAGE_SWITCH_K=3 在短 PPT（<10页）和长 PPT（>30页）上的表现是否一致？

## 决策流程说明（给未来的自己）

```
测试平台验证策略
      ↓
在上面对应模块的表格里更新状态和参数值
      ↓
修改对应 services/*.py 文件里的常量或逻辑
      ↓
后端 API 自动使用新策略（无需改 API 层）
      ↓
前端用户拿到的结果就是最新策略的输出
```

**核心原则**：策略（算法参数、prompt）在 `services/` 和 `prompts/` 里改，API 层（`routers/`）和前端不需要动。
