# AI 产品面试思考框架

## 第一步：把项目分清层级

所有 AI 产品项目讲述，本质也就三层：

| 层级 | 核心问题 | 常见误区 |
|------|----------|----------|
| **第一层：功能层** | 你实现了什么 AI 能力（识别、生成、推荐等），是否跑通 demo | 只停留在这层，面试官一追问就露馅 |
| **第二层：设计层** | 怎么做需求定义、模型选型、数据策略、效果评估指标怎么定、badcase 怎么分析 | 说不清选型理由，说不清指标从哪来 |
| **第三层：工程与交付层** | 系统稳定性怎么保障，推理延迟怎么优化，线上 badcase 怎么兜底，用户反馈怎么闭环 | 没上线经验的人这层完全空白 |

很多候选人只停留在第一层。面试官一追问，就发现你只是"调过接口"，不是"设计过产品"。

---

## 第二步：每个技术/产品点准备"讲法结构"

以"模型选型"为例：

**常见（差）讲法：**
> "我用了 GPT-4，效果不错。"

这句话信息量很低，听不到你做产品决策的过程。

**面试官更想听的是：**
- 业务对效果和成本的核心要求是什么？
- 你怎么评估不同模型（GPT-4、Claude、Llama、国产模型）在具体场景下的表现？
- 准确率、召回率、响应延迟、成本、数据安全，你怎么权衡？
- 为什么选大模型而不是小模型微调？
- 评测集怎么构建？badcase 怎么分析？
- 上线后效果和预期相比如何？怎么迭代？

---

## 第三步：会做技术/产品对比的人加分很多

很多同学讲选型只会一句："这个模型比较主流。"——这属于没讲。

面试官更想听你怎么对比、怎么取舍：
- 为什么选 GPT-4 而不是 GPT-3.5 或 Claude？
- 为什么用 RAG 而不是微调？

## 附：LiberStudy 的 AI 能力图谱（第一层展示素材）

### 核心 AI 流水线

```
音频 → [ASR] → 精校转录稿 → [语义对齐] → 按页分段 → [LLM] → 结构化笔记
                                    ↑
                          PPT 文字 + 用户锚点
```

### P0 核心能力（MVP 已实现）

**1. ASR — 语音识别**
- 中文：阿里云智能语音 API；英文：OpenAI Whisper API
- 输出带时间戳转录稿，再做 filler word 过滤 + 标点重建
- 质量目标：中文 WER ≤ 10%

**2. 语义对齐 — text-embedding-3-small**
- 转录稿片段 × PPT 页文字双向 embedding，余弦相似度匹配
- 辅以用户停留锚点和手动标注锚点（三类锚点优先级机制）
- 严禁均分时长，支持非线性教学流

**3. 被动学习笔记生成 — Claude claude-sonnet**
- 不管用户有没有记笔记，每页 PPT 自动生成结构化注释
- JSON Mode 强制结构化输出，严格限制只用转录内容（防幻觉）
- 成本约 $0.13 / 20 页 PPT

**4. 主动学习笔记扩写 — Claude claude-sonnet**
- 用户有记笔记的页面，LLM 把关键词扩写成完整知识点
- 在被动笔记基础上叠加，不替换
- 4 种笔记模板 × 详略两档粒度切换（模板切换复用缓存，不重新调用 LLM）

### P1 规划中（未实现）

- **Quick Ask 实时问答**：录音中浮窗，滚动 30 分钟 transcript buffer，回答"老师刚才说了什么"
- **无 PPT 模式**：只用转录稿，LLM 自动按话题段落组织笔记
- **图片页 OCR 兜底**：PyMuPDF 提取不到文字时，走 VLM 补充

### V2 延后

- RAG 跨节课知识问答
- 自动生成复习卡片
- 视频帧检测 PPT 翻页（pHash → SSIM → CLIP fallback）

---

### 核心技术赌注（用于回答"最大的技术风险是什么"）

PRD 明确定义了唯一的技术生死线——**假设 B**：

> ASR 转录 + PPT 内容语义对齐 + LLM 笔记生成的质量是否足够好

其他所有 AI 能力都是锦上添花，这条主链路是决定产品有没有价值的根本。MVP 的首要目标就是验证假设 B。
