# LiberStudy MVP-0 功能实现进度（2026-04-10）

## 已实现（可运行）

### 输入采集
- 麦克风实时录音（开始/暂停/继续/停止）
- 音频每 10 分钟分片存储到 IndexedDB
- PPT 文件上传（拖拽+点击，支持 .ppt/.pptx/.pdf，≤100MB）
- 就地文本批注（点击 PPT 画面弹出输入框，记录页码+时间戳+Y轴位置）
- 右栏专属笔记区（可直接输入）
- 无 PPT 两栏布局模式（自由文本输入区）

### 会话中断恢复
- 音频切片持久化 IndexedDB
- 页面重新打开时弹出恢复对话框（三个选项：继续/生成/放弃）

### 后端处理流程
- PPT 解析：LibreOffice → PDF → PyMuPDF 提取文本 + 渲染 PNG
- 音频格式转换：FFmpeg（任意格式 → 16kHz mono WAV）
- ASR 转录：OpenAI Whisper（含时间戳）+ 口头语过滤
- 语义对齐：`text-embedding-3-small` 计算相似度矩阵，自动分配逐字稿片段到 PPT 页
- 笔记生成（Claude claude-sonnet-4-6）：被动学习（所有页）+ 主动学习（有批注页），并发 ≤5，重试 3 次
- `page_supplement`（脱离课件内容独立收纳）
- 单页重试 API 端点（后端已有）

### 笔记查看界面
- 三栏布局（大纲导航/PPT画布/笔记区）
- 药丸型切换控件（"我的笔记" / "AI 笔记"）
- 简单/详细粒度切换（简单模式隐藏 AI 注释）
- 时间戳点击跳转音频播放
- 对齐置信度低标注 ⚠️（< 0.6）
- 音频播放器（播放/暂停，±10s 快进/快退，进度条）
- 导出 Markdown（完整会话 .md 下载）
- 一键复制当前页笔记

### 其他
- 处理进度界面（4 阶段进度条）
- 限流（每 IP 每天 2 次）
- Mock 会话（3 页，覆盖各种笔记场景）
- 端到端集成测试

## 未实现 / 存在缺陷

### 关键功能缺口（影响主流程）

| 功能 | PRD 优先级 | 问题描述 |
|------|-----------|---------|
| **前端调用真实接口** | P0 | `uploadFiles()` 仍调用 `/api/process-mock`，真实 `/api/process` 存在但前端未接入 |
| **ProcessingPage 不轮询后端** | P0 | 进度条是写死的动画，不读取真实处理状态，不知道后端何时真正完成 |
| **IndexedDB 音频切片未上传** | P0 | `RecordingControl` 存了切片，但生成笔记时只发单个 Blob，不从 IndexedDB 读取/合并切片 |
| **用户批注未传给后端** | P0 | SessionPage 采集的批注（`user_anchors`）没有序列化发给 `/api/process`，对齐完全不用用户锚点 |
| **阿里云 ASR** | P0 | `transcribe_aliyun()` 仅为 stub，抛 `NotImplementedError`，中文 ASR 实际回落到 Whisper |
| **会话数据仅存内存** | P0 | 无数据库，后端重启所有会话丢失 |

### 界面功能缺口

| 功能 | PRD 优先级 | 问题描述 |
|------|-----------|---------|
| **4 种 AI 笔记模板差异渲染** | P0 | `TemplateSelector` 下拉有 4 种模板，但 `NotesPage` 忽略选择值，全部渲染相同内容 |
| **单页重试按钮未接线** | P0 | `partial_ready` 页面显示"点击重试"按钮，但无 `onClick`，`retryPage()` API 从未被调用 |
| **讲解不足 10 秒页面标注** | P0 | PRD 要求标注"讲解内容较少，建议参考上下页"，未实现 |
| **大段静默跳过** | P0 | PRD 要求 > 2 分钟静默不生成笔记，未实现 |

### P1 功能（尚未开始）

## 总结

核心处理链路（后端 4 步 pipeline）已完整，但**前后端联调未完成**。
最大阻塞项：
1. 前端仍调 Mock 接口
2. 用户批注不传后端
3. 处理进度不轮询真实状态
